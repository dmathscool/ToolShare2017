Notes:
I have implemented separate tables for your owned items and items you are borrowing. Otherwise we have to come up with some way to indicate, for example,
whether a tool is on loan to you or you have loaned it out.

"Next step" on buttons should automatically show something more descriptive based on the item status.
Suggesting items with status Requested show "Accept" on the button, Accepted shows "Deliver", On loan shows "Reclaim", Returned shows "Rate user"
Not all items will necessarily need a button. Most of the ones on "their tools" probably won't. The tool owner will typically drive the workflow.
The one exception I can think of is rating the loaning user after you have returned their item.

When the loaning user accepts the request, there are probably two simplest ways to handle the hand off:
1) system automatically emails both users a notice that says, for example:
"tool X is ready to trade. owner: Bob Jones (bjones@gmail.com). borrower: Susan Smith (ssmith@yahoo.com). coordinate, hand off, and mark the tool loaned"
2) a notes field in the transaction list shows e-mail address of the other user with the intent that the users just e-mail each other to coordinate handoff.

We also need an expiration date column
